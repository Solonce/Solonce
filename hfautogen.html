HFAutogen Release:

Quick zero-cost protoyping of multi-agent architecutres.

What is HFAutogen?
HFAutogen is a python package that allows the user to utilize multi-agent communication for little to no processing power and free of charge.

It acheives this by leveraging HuggingFace's Serverless Inference API and having Custom Model Clients provided by Autogen to accept non-OpenAI formatted requests. This allows a user to acheive omplex conversations between different agents with different intent or use-cases.
Why HFAutogen?
Simple & Fast Prototyping:
Quickly prototype free multi-agent communication structures with low-memory machines.

Multiple Agents:
Synthesize Programmers, Project Leads, Analysists, Experts, Test-Users, etc. At the time on release, HFAutogen only supports two agents. The next release will include GroupChat functionality. See 'Restrictions.'

Autonomous Code Generation & Execution:
Generate, execute, debug, and save scripts to be used by an agent or to a user.

How to Install
Install HFAutogen using pip as seen below.
pip install hfautogen
How to Use
You can find the documentation on the Github page here.
from hfautogen import ModelAgent, UserAgent, InitChat

_input = input("Write me a simple poems about flowers\n")
hf_key = "hf_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

user = UserAgent("user_proxy")
assistant = ModelAgent("assistant",
                        hf_key,
                        system_message="You are a friendly AI assistant.")

InitChat(user, assistant, _input)
There are three main functions you want to use.

UserClient
An agent that acts as the user, that uses a model provided by HuggingFace's free inerference api to generate requests to the ModelClient()

ModelClient
An agent that uses a model provided by HuggingFace's free inference api to respond to the UserCleint() requests/comments.

InitChat
A function that allows a UserClient() and a ModelClient to interact with one another.
Capabilities
Two-Agent Communication:
At the time of this projects release, it is only able to interact between a user agent, and a model agent. See 'Restrictions' for the current limitations this provides.

Quick Prototyping:
HFAutogen allows quick protyping of complex agent architectures. The free access to an api allows little to no computing power required with no cost associated. This allows for beginners and expert alike to avoid the usual overhead assiciated with AI development.

Automatic Code Execution:
The user agent is able to execute and save code. This means that you are able to autonomously create scripts that can acheive a given task. Given the correct input, one could create scripts to replicate human actions in a solo or team based setting. Allowing quick protoyping of different work environment.

Restrictions
GroupChat features normally found in Autogen are not currently implimented, restricting the complexity of conversations that one can produce.

Not all models found on HuggingFace will be available for use for free, only those that have serverless inference already set up.

In Progress
The biggest feature that is a work in progress is implimenting GroupChat funcitonality. This will allow a user to create entire teams of agents that work in conjunction toward a common goal.

Displaimer & Explination
Disclaimer:
I created and had no part of the creation of Autogen or HuggingFace's Inference API. I simply made the two compatible to work together, which was not previously a feature.

Explination:
The code that enables HFAutogen to work is relatively simple. Autogen has two methods, CustomModelClient and CustomModelClientWithArguments. These methods allow me to create a custom model client that takes in a HuggingFace Inference API link, rather than strictly an OpenAI compatible link. Then, because the data received is not in an OpenAI compatible format, it handles the processing and contextulizing of that data for the agents to properly use.
